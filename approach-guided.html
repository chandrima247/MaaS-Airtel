Approach 2 · Guided Conversational Stepper

Guide the user one decision at a time.

This version is the controlled middle path. It asks only the most important questions in sequence, creates a profile object for the user, and then hands them into listing with better personalization and lower ambiguity.

High clarity Best for health Lower interpretation risk

Why this is the core flow

Health is not like motor or screen cover. Segment drives product mapping, so the stepper starts with who needs protection, then narrows into product type, coverage logic, and preferences.

Child → J-Junior Parent → Senior / cash Family → J-Care / similar

Live profile object

The flow builds this object before listing, which means pricing, product ranking, and filters already know the user context.

{
  "segment": "child",
  "type": "comprehensive",
  "age": 8,
  "members": 1,
  "coverage": "30M",
  "priority": "balanced"
}

1. Who do you want to protect?

Start with segment because that determines product mapping before price, benefits, or hospital network.

2. What kind of cover do you want?

Keep this simple. Users usually understand full medical vs hospital cash much faster than a long list of product names.

3. Add the core inputs

Ask only what is required for ranking and quoting. Everything else can be optional or handled later.

4. Optional preferences

These are useful, but not mandatory. This is where progressive reveal helps lower drop-off.
